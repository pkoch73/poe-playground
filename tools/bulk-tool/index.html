üéØ Overview

The Assets Bulk Tool is a powerful web application designed to perform bulk operations on Adobe Experience Manager (AEM) assets. It provides four main operations to help you manage your Digital Asset Management (DAM) efficiently.

üìù Asset Renamer

Bulk rename asset files by replacing or removing specific characters from filenames.

üìã Metadata Updater

Update asset metadata properties using data from Excel spreadsheets.

üóëÔ∏è Bulk Delete

Permanently delete multiple assets from your DAM in one operation.

üîç Bulk Search

Find and download assets by SKU or metadata criteria with filtering options.

üîß Requirements & Setup

System Requirements

  • Modern web browser (Chrome 90+, Firefox 88+, Safari 14+, Edge 90+)
  • JavaScript enabled
  • Access to Adobe Experience Manager (AEM) instance
  • Valid AEM user credentials or Adobe IMS authentication

Supported File Formats

  • Excel files: .xlsx, .xls
  • CSV files: .csv
  • Maximum file size: 10MB

AEM Permissions Required

‚ö†Ô∏è Important: Your AEM user account needs the following permissions:

  • Read: Access to /content/dam and asset metadata
  • Write: Modify asset properties and metadata (for rename/metadata operations)
  • Delete: Remove assets (for delete operations)
  • Query: Search assets using Query Builder API

üîê Authentication

The tool supports three authentication methods:

1. Basic Authentication

Standard username/password authentication for local AEM instances.

  • Best for: Local development, author instances
  • Required: AEM host URL, username, password
  • Example: admin/admin for local instances

2. Adobe IMS Authentication

OAuth-based authentication for AEM Cloud Service instances.

  • Best for: AEM as a Cloud Service, production environments
  • Required: Adobe ID account with appropriate permissions
  • Automatic token retrieval via popup window

3. Manual Token Entry

For testing or when automatic authentication fails.

  • Best for: Testing, troubleshooting, custom setups
  • Required: Valid IMS access token
  • Can be obtained from browser developer tools or Adobe I/O Console

‚öôÔ∏è Available Operations

üìù Asset Renamer

Purpose: Bulk rename asset files by replacing or removing specific characters.

How it works:
  1. Upload Excel file with asset paths in one column
  2. Select characters to process (underscore, space, hyphen, period)
  3. Choose to replace with another character or remove completely
  4. Preview changes before execution
  5. Execute renames using AEM Assets HTTP API
Example use cases:
  • Replace underscores with hyphens: "product_image.jpg" ‚Üí "product-image.jpg"
  • Remove spaces from filenames: "my image.jpg" ‚Üí "myimage.jpg"
  • Standardize filename conventions across your DAM

üìã Metadata Updater

Purpose: Update asset metadata properties using data from Excel spreadsheets.

How it works:
  1. Upload Excel file with asset paths and metadata values
  2. Map Excel columns to AEM metadata properties
  3. Add multiple property mappings as needed
  4. Preview metadata changes with HTTP request details
  5. Execute updates using AEM Assets HTTP API
Common metadata properties:
  • dc:title - Asset title
  • dc:description - Asset description
  • dc:subject - Keywords/tags
  • jcr:lastModified - Last modified date
  • cq:tags - AEM tags

üóëÔ∏è Bulk Delete

‚ö†Ô∏è WARNING: Delete operations are permanent and cannot be undone!

Purpose: Permanently delete multiple assets from your DAM.

How it works:
  1. Upload Excel file with asset paths to delete
  2. Review the list of assets to be deleted
  3. Confirm the deletion operation
  4. Execute deletions using AEM Assets HTTP API
Safety recommendations:
  • Always backup your AEM instance before bulk deletions
  • Test with a small subset first
  • Verify asset paths are correct
  • Coordinate with your team before executing

üîç Bulk Search

Purpose: Find and download assets by SKU or metadata criteria.

How it works:
  1. Upload Excel file with SKUs or search terms
  2. Configure search criteria (metadata property, base path, limits)
  3. Execute search using AEM Query Builder API
  4. Filter results by asset type, product shot, etc.
  5. Select and download asset manifest
Search features:
  • Search in multiple metadata properties
  • Filter by asset type and product shot
  • Bulk select and download
  • Results grouped by SKU
  • Thumbnail previews (when authenticated)

üìä Excel Sheet Structure Guide

üìã General Requirements for All Excel Files

  • Header Row: First row must contain column headers (descriptive names)
  • Data Rows: Start from row 2, no empty rows between data
  • File Format: .xlsx, .xls, or .csv (Excel preferred for better compatibility)
  • File Size: Maximum 10MB
  • Encoding: UTF-8 for special characters
  • Empty Cells: Leave cells empty rather than using "N/A" or dashes

üîó Asset Path Format

Asset paths must follow the correct AEM DAM structure:

‚úÖ Correct formats:
/content/dam/products/laptop.jpg
/content/dam/marketing/banners/summer-sale.png
/content/dam/documents/manual.pdf

‚ùå Incorrect formats:
laptop.jpg (missing path)
products/laptop.jpg (missing /content/dam)
\content\dam\products\laptop.jpg (wrong slashes)

üí° Tips for Asset Renamer:
  • Use any column header name (e.g., "Asset Path", "File Path", "DAM Path")
  • Include only assets that need renaming
  • Verify paths exist in your AEM instance before running
  • Test with a small batch first

üìã Metadata Updater - Excel Structure

Required Columns:
  • Asset Path: Full path to the asset in AEM DAM (required)
  • Metadata Columns: One or more columns with metadata values (flexible naming)
Example Structure:
Asset Path Title Description SKU Keywords
/content/dam/products/laptop.jpg Gaming Laptop Pro High-performance gaming laptop with RGB keyboard LAP-001-RGB gaming, laptop, RGB, performance
/content/dam/products/mouse.jpg Wireless Gaming Mouse Ergonomic wireless mouse for gaming MOUSE-WL-01 mouse, wireless, gaming, ergonomic
Common AEM Metadata Properties:
Dublin Core (dc:)
  • dc:title - Asset title
  • dc:description - Asset description
  • dc:subject - Keywords/subjects
  • dc:creator - Creator name
  • dc:rights - Copyright info
Custom Properties
  • sku - Product SKU
  • productId - Product ID
  • brand - Brand name
  • category - Product category
  • cq:tags - AEM tags
üí° Tips for Metadata Updater:
  • Column headers can be descriptive (e.g., "Product Title" maps to dc:title)
  • Empty cells will set the property to empty (not skip the update)
  • Use comma-separated values for multi-value properties
  • Special characters are preserved (UTF-8 encoding)
  • You can have additional columns that won't be used

Do's

üîß Troubleshooting

Authentication Issues

  • 401 Unauthorized: Check credentials, verify user permissions
  • IMS popup blocked: Allow popups for this site
  • Token expired: Re-authenticate or get a new token
  • CORS errors: The app uses a CORS proxy - check network connectivity

File Upload Issues

  • File too large: Maximum 10MB, split large files
  • Unsupported format: Use .xlsx, .xls, or .csv files only
  • Empty sheet: Ensure your Excel sheet has data
  • No headers: First row should contain column headers

Operation Failures

  • 404 Not Found: Asset path doesn't exist, check spelling
  • 403 Forbidden: Insufficient permissions for the operation
  • 500 Server Error: AEM instance issue, check server status
  • Network timeout: Large operations may take time, be patient

Performance Tips

  • Batch size: Process 50-100 assets at a time for best performance
  • Network speed: Use wired connection for large operations
  • AEM load: Avoid peak usage times when possible
  • Browser memory: Close other tabs for large operations

üîå API Details

The tool uses standard AEM APIs for all operations:

Assets HTTP API

  • Base URL: /api/assets
  • Rename: MOVE method with X-Destination header
  • Metadata: PUT method with JSON payload
  • Delete: DELETE method

Query Builder API

  • Endpoint: /bin/querybuilder.json
  • Method: GET with query parameters
  • Used for: Asset search and discovery
  • Returns: Asset metadata and paths

CORS Proxy

  • Purpose: Bypass browser CORS restrictions
  • URL: https://cors-proxy.philipp-koch.workers.dev/
  • Security: Authentication headers are forwarded
  • Limitation: Third-party service dependency

üõ°Ô∏è Security Considerations

  • All operations require proper AEM authentication
  • CORS proxy is used for cross-origin requests
  • No credentials are stored locally or transmitted to third parties
  • All API calls follow AEM security best practices
